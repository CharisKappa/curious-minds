Chapter 02 · The mechanics of learning

How a Model Learns

There is no programmer writing rules for what a language model does. There is a number. The number measures how wrong the model is. Training is the process of making that number smaller — billions of times, over billions of examples. This chapter shows exactly how that works.

~42 min read 11 sections The mechanics of learning

What learning means here

When we say a child learns to recognise a cat, we mean something messy and hard to pin down — exposure, feedback, gradual refinement of some internal representation we can't directly inspect. When we say a model learns to recognise a cat (or complete a sentence, or write code), we mean something very specific and fully mechanical: the adjustment of numerical values inside the model, guided by a mathematical signal, until the model's outputs get closer to the desired outputs.

There is no mystery here. There is no ghost in the machine. What looks like understanding is the result of a process you could, in principle, follow step by step — it would just take you several trillion steps longer than a GPU cluster takes.

That said, the gap between "fully mechanical" and "we understand exactly what's happening" is larger than it looks. We can describe the training process precisely. We cannot fully explain why training a model on text prediction produces something that can reason about ethics, write poetry, or debug code. The mechanics are clear. The emergent behaviour is not. This tension runs through everything in AI right now.

The neural network as a machine

A neural network is a mathematical function — a very large, very nested mathematical function. It takes numbers in, does a lot of arithmetic, and produces numbers out. That's it.

The "neural" part refers to the loose inspiration from neuroscience: the basic unit is a neuron (also called a node or unit), which takes several inputs, multiplies each by a weight, adds them together, and passes the result through a simple function. Neurons are organised into layers. The output of one layer feeds into the next. A "deep" network is simply one with many layers.

The genius of this architecture — and the reason it works at all — is that it's a universal approximator. Given enough neurons and enough layers, a neural network can approximate any function, including the extremely complex function that maps "a sequence of words" to "the next most likely word." You don't design the function. You design a structure capable of learning it.

Parameters: the knobs

Every connection between neurons has a weight — a number. Every neuron has a bias — another number. These are the parameters of the model. A small model might have a few million. GPT-3 has 175 billion. GPT-4 is estimated at over a trillion. Llama 3 70B has — as the name suggests — 70 billion.

Before training begins, these parameters are initialised to small random values. The model at this point is useless — it produces random outputs for any input. Training is the process of adjusting these billions of numbers, iteratively, until the model's outputs are consistently good.

Parameter
A single adjustable number inside a neural network. During training, every parameter is nudged slightly in a direction that reduces the model's error. A model described as "7B" has 7 billion such numbers. More parameters = more capacity to learn complex patterns, but also more compute and memory required.

The forward pass

Take a concrete example. We want to train a model to predict the next word in a sentence. We feed it the text: "The cat sat on the ___". The model needs to predict "mat" (or something plausible).

The text is first converted to numbers — each word or word-fragment (a token) is represented as a number from a fixed vocabulary. These numbers are fed into the network layer by layer. Each layer transforms the data — detects patterns, combines information. At the end, the model outputs a probability distribution over all possible next tokens. Maybe it assigns 40% probability to "mat," 15% to "floor," 10% to "chair," and so on.

This is the forward pass: input goes in, prediction comes out. It takes a fraction of a second and involves billions of multiplications and additions.

Loss: measuring wrongness

We know the correct answer: "mat." The model predicted "mat" with 40% probability. Is that good or bad? We need a precise, mathematical way to measure how wrong the model is. That measure is called the loss (or loss function, or cost function).

For language models, the standard loss function is cross-entropy loss, which essentially asks: "how surprised was the model by the correct answer?" If the model assigned 40% probability to "mat" and "mat" was correct, it was somewhat surprised — loss is moderate. If it assigned 1% probability to "mat," it was very surprised — loss is high. If it assigned 95%, it was barely surprised — loss is low.

Training has one goal: make the average loss across all training examples as small as possible. Everything else — the architecture choices, the data curation, the compute budget — is in service of this single objective.

Backpropagation

We have a loss number. We know the model was wrong by this much. Now what? We need to adjust the parameters to make the model less wrong. But there are billions of parameters. How do we know which ones to adjust, and in which direction?

The answer is backpropagation — one of the most important algorithms in modern computing. After the forward pass produces a prediction and we calculate the loss, backpropagation runs the calculation in reverse. Using the chain rule from calculus, it computes for each parameter: "if I increase this parameter slightly, does the loss go up or down, and by how much?"

This gives us a gradient for every parameter — a direction and magnitude. The gradient tells us how to adjust each number to reduce the error.

Gradient descent

Armed with gradients, we update every parameter by a small amount in the direction that reduces loss. This is gradient descent. The "descent" refers to descending the loss landscape — imagine a hilly terrain where every point represents a possible set of parameter values and height represents loss. Gradient descent is how you roll downhill toward a valley.

The size of each update step is controlled by the learning rate — one of the most important hyperparameters in training. Too large and the model overshoots — it bounces around without settling. Too small and training takes forever. Getting the learning rate right (and adjusting it over the course of training) is one of the core practical skills of training models.

The training loop

Put it all together and you have the training loop:

1. Take a batch of training examples (maybe 1,000 text sequences). 2. Run each through the network (forward pass). 3. Calculate the loss for each example. 4. Average the losses across the batch. 5. Run backpropagation to get gradients. 6. Update every parameter using those gradients. 7. Repeat.

For a large language model, this loop runs billions of times. GPT-3 was trained on roughly 300 billion tokens of text — each token processed multiple times during training. The training run took months on thousands of GPUs running simultaneously. The loss, which started high (the model was guessing randomly), gradually fell as the model got better at predicting the next token.

Why predicting the next word produces something useful

It might seem like "predict the next word" is a trivial task with a trivial objective. It is not. To predict what word comes next in any arbitrary sentence, a model must implicitly learn grammar, facts, reasoning patterns, writing styles, logic, and a vast amount of world knowledge — because all of these affect what words typically follow what other words. The task is simple. What you have to learn to do it well is not.

Why scale changes everything

This training loop is not new. Neural networks trained with backpropagation and gradient descent have existed since the 1980s. What changed was scale — and scale changed the outcome qualitatively, not just quantitatively.

Small models trained on small data learn surface-level patterns. They can predict likely word completions in a narrow domain. Large models trained on vast data do something different: they appear to develop internal representations of concepts, facts and reasoning strategies. A model trained on enough text doesn't just memorise word sequences — it builds something like a compressed model of the world that generated that text.

This is why researchers talk about emergent capabilities — behaviours that appear in large models but not small ones, that weren't explicitly trained for and that surprised their creators. A model trained purely on next-token prediction can write code, translate languages it wasn't explicitly trained on, and perform multi-step reasoning. Why? We don't fully know. The training objective is simple. What it produces is not.

Pre-training vs fine-tuning

What we've described so far — training a model to predict next tokens on a massive text corpus — is called pre-training. It produces a base model — a model with vast knowledge and capability but no particular personality or alignment. Ask it to complete "The best way to make a bomb is" and it might, because it's just predicting likely next tokens based on text it saw during training.

To make a model useful and safe, a second training phase is applied: fine-tuning. Fine-tuning uses the same gradient descent process but on a much smaller, carefully curated dataset of examples showing the model how to behave — how to answer questions helpfully, how to refuse harmful requests, how to follow instructions. This is combined with techniques like RLHF (Reinforcement Learning from Human Feedback) to align the model with human preferences. The result is what you interact with when you use ChatGPT or Claude.

What the model actually knows

A trained model is, in a sense, a compression of its training data. All those billions of parameters encode patterns extracted from trillions of words of text. But it's not a database — you can't look up "what did the model learn about Paris" and get a clean answer. The knowledge is distributed across billions of parameters in ways that are not human-readable.

This has important practical consequences. Models sometimes confidently state things that are wrong — not because they're lying, but because the pattern-matching process that produces plausible-sounding text sometimes generates plausible-sounding wrong text. This is called hallucination, and it's one of the fundamental challenges of current language models. The model's training objective was to produce plausible next tokens, not to produce truthful ones. These are usually the same thing. Sometimes they're not.

The strangeness of what you've built

Step back and notice what's strange about this. You started with a network of random numbers. You ran a mechanical optimisation process on billions of text examples. You ended up with something that can write essays, pass medical exams, explain quantum mechanics and compose poems. Nobody programmed any of those capabilities. They emerged from gradient descent on next-token prediction at sufficient scale. This is either the most impressive result in the history of computer science or an uncomfortable sign that we don't understand what we've built. Probably both.
